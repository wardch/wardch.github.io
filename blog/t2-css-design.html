The Differences between Relative, Fixed and Absolute Positioning.

I'll try offer some insight into the difference between CSS positioning types, namely relative, fixed and absolute.

15/02/2015

Relative positioning is relatively easy to understand! If you type in a css file: h1 { position:relative; left: 50px; } It means that the elements between all of the h1 tags in your HTML file, will appear 50px to the left of where they would have appeared otherwise!

Absolute Positioning

These elements are positioned on a page in relation to their "parent" element. So what is a parent element? It is the element that is above the element of a "child element." A HTML element can have one parent but many children. Let me show you an example through code to help clear up what I mean.

Let's say I have two lines of HTML code. The first line is a div which is a wrapper. This means that the wrapper will take up almost the entire browser screen if we set it's width to 95%. Now let's say we have an element which will go within our wrapper, like a h1 tag. If we use absolute positioning and in a css file type h1 { display: absolute; margin-left: 5em; } It means that h1 will have a margin of 5em from it's parent element. What is the element which h1 is within? In this example it's the wrapper element.

Fixed Positioning

Imagine if the parent element of a child element was the browser window. Now you have an idea of what fixed positioning is. Fixed positioning fixes and element to a browser window. The element will not move up or down as the user scrolls up and down. This is useful for when a coder may be making a website which has a header that will remain on the page, no matter how much a user scrolls down.
