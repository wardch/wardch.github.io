Javascript Prototypes

So, what exactly is a Javascript prototype?

For this cheat sheet we're going to go over Javascript prototypes.Javascript doesn't share classical inheritance in the way that Object oriented languages do (with classes, parent classes, child classes, etc.) Nevertheless, inheritance is still present in Javascript thanks to the concept of the 'prototype'. In order to follow DRY programming principles, Javascript has created a simple way to add, update, inherit, and lookup properties of linked sets of Objects. The mechanism that is used to perform all of these actions is the 'constructor' function, which relates to the JS Object's prototype.

Broadly speaking, prototype is a Javascript method inherent in all Javascript objects.

Let's walk through an example to really flesh out how it works:

function Animal(name, numLegs) {
        this.name = name;
        this.numLegs = numLegs;
    }

First we are creating a new animal constructor function, which takes in two parameters, and assigns those two parameters to properties, by the name of "name" and "numLegs", which are properties of "this" object which we are in. (which refers to a function object, named Animal).

How does the prototype affect inheritance?

If we want to add properties or methods to the Animal constructor, we use the prototype property. Prototype will allow us to alter the state of the animal function that we have created. If we add functions to our prototype, they're now known as "methods". Here we have added a "sayName" method to the Animal object. 

Animal.prototype.sayName = function() {
      console.log("Hi my name is " + this.name);
    };

    Animal.prototype.heartbeat = true;

Now we'll demonstrate how a newly made constructor function can inherit methods and properties from a preexisting constructor function(animal). 

// define a Penguin class
    function Penguin(name) {
        this.name = name;
        this.numLegs = 2;
    };

We create the penguin object, and then use the code snippet below to move Animal's methods and properties into our new Penguin object. 

// set its prototype to be a new instance of Animal
    Penguin.prototype = new Animal();

Inheritance, via Prototypes

Now every penguin will be known as a new animal on top of being it's own new Penguin. It's like inheritance after the fact. Just like in classical inheritance prototype properties and methods will override equivalently named animal elements that go by the same name. In this case for example, all Penguins numLegs are set to a default of two, even if we pass in 4 legs. 

var penguin = new Penguin("Jimbo");

  // penguin.sayName

  var penguin2 = new Penguin("LeeRoy", 4);
  // => { name: 'LeeRoy', numLegs: 2 };

Even though we have a numLegs property in our Animal object, which our Penguin object inherited, our Penguin object has a property that over-wrote the numLegs property. This mimics classical inheritance 

console.log(penguin2.heartbeat);
    // => true

This was inherited from the animal object, since the Penguin object didn't have a preexisting heartbeat property.
